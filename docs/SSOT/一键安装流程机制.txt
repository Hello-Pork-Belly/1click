一键安装流程机制（1click A0 规范版）

目标
- 提供一条可重复、可回滚、可审计的一键安装执行路径。
- 把“需求 -> 计划 -> 执行 -> 验证 -> 发布”串成可证据化流水线。

流程总览
1) 现实校验（Reality Check）
- 先确认远端主干、PR、Actions、Release 现实状态。
- 所有结论必须附可点击 URL。

2) 计划收敛（Plan）
- 明确本次任务目标、非目标、文件白名单、风险与回滚。
- 若为 docs-only 任务，禁止触达业务代码与 workflow 逻辑。

3) 执行（Implement）
- 仅修改白名单内文件。
- 任何超出白名单的改动都视为失败并回退到计划重审。

4) 审计（Verify）
- 运行最小验证命令（优先 `make check`）。
- 记录命令、关键日志、退出码。
- 执行 RULES 禁止项扫描（archive/upstream/snapshot 等）。

5) 交付（Release/Merge）
- 创建单一主题 PR，附完整 Evidence Pack。
- 合并后回填主干最新 SHA 与 merge commit URL。

证据要求
- main HEAD URL
- PR URL / head SHA / base SHA
- checks 或 no-checks 证据
- actions run URL（或 none 证据）
- merge commit URL
- post-merge main HEAD URL

失败处理
- 任一硬门禁失败：标记 FAIL。
- 在 STATE 的 Next 写补救任务（Task ID、具体动作、所需远端证据）。
